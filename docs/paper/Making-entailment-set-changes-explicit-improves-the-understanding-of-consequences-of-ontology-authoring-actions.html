Abstract訳

公理を追加したり削除したりした結果は，OWLを使ってオントロジーオーサリングを把握する際に難しい．モデリング行為の結果は意図しない推論から非一貫性(incoherency とinconsistency)のような明らかな欠陥まで幅広くある．オントロジーオーサリングにおける中心的な活動の一つは，特定 のモデリングステップが意図した結果を生み出すか，よく推論機構の力を借りながら，を確認することである．例えば，protegeのユーザにとっては，例えば，これは推論されたクラス階層を探索することを含んでいる．
私たちは，鍵となる論理的含意の集合に対する変化明らかにすることは標準的な静的な階層とかフレームベースの方法と比べて確認作業を効率化するという仮説を立てて検証する．私たちはその方法をprotegeのプラグインとして実装し，探索的な実験を行った．その実験は私たちの方法からユーザにとって役に立つオーサリング行為を分離するためのものである．次に私たちは私たちの仮説を検証し，鍵となるオーサリング問題に対して，論理的含意の集合を明示的に変化させることが正確性と速さの両方から結果の理解に貢献することを見つけるための，コントロール実験を行った．そして，通常の見え方に比べて変化を追うのに良い方法であると評価された．
